## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps within plugin cards |
| sm | 8px | Inline padding inside buttons, gap between nav links |
| md | 16px | Default padding inside cards, horizontal padding in header |
| lg | 24px | Section padding-y, gap between plugin cards in grid |
| xl | 32px | Hero section vertical padding, gap between hero columns |
| 2xl | 48px | Major section breaks between Hero / Plugin Showcase / Pricing |
| 3xl | 64px | Top padding on first hero section (below sticky header) |

Container: `max-w-screen-xl mx-auto px-8` — established pattern from Footer.tsx, applies to all sections.

Exceptions:
- Sticky header height: 56px (not a spacing token — set via `h-14` Tailwind class)
- Phone device frame inner area: ~340px wide, ~680px tall (proportional to iPhone 14 Pro — CSS-only)
- Touch target minimum: 44px height on all CTA buttons (accessibility requirement)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Plugin descriptions, pricing card value props, nav links |
| Label | 14px | 400 (regular) | 1.4 | Footer links, copyright, secondary meta text (already established in Footer) |
| Heading | 20px | 700 (bold) | 1.3 | Plugin category headings, pricing card title, section headings |
| Display | 36px (mobile: 28px) | 700 (bold) | 1.15 | Hero headline, pricing price display ("0€") |

Font family: Inter (self-hosted). Exactly 2 weights declared: 400 (regular) and 700 (bold).

Responsive display: Hero headline uses `text-3xl` (30px) on mobile, `text-4xl` (36px) on `md:` breakpoint and above.

## Color

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `#F7F6F3` (`bg-background`) | Page background, hero section background | FOUND-03 / globals.css |
| Secondary (30%) | `#FFFFFF` (`bg-white`) | Plugin cards, pricing card, sticky header background | Footer.tsx pattern |
| Accent (10%) | `#FF5C1A` (`text-primary` / `bg-primary`) | See reserved-for list below | FOUND-03 |
| Muted text | `#6B6963` (`text-muted`) | Plugin descriptions, subline text, header secondary text | globals.css |
| Text default | `#1C1A17` (`text-text`) | Headings, plugin names, nav links, pricing copy | globals.css |
| Border | `#E2E0DA` (`border-border`) | Plugin card borders, pricing card border, header bottom border | globals.css |
| Destructive | not applicable | No destructive actions in this phase | — |

**Accent (`#FF5C1A`) is reserved exclusively for:**
1. App Store CTA button background (Hero)
2. Play Store CTA button background (Hero)
3. "Download for free" CTA button background (Pricing)
4. "Télécharger l'app" / "Download the app" button in sticky header
5. Header bottom border (1px — matches Footer top border pattern)
6. Hero subline icon accent (Ionicons checkmark icons, if used in value prop list)
7. Logo text "Ziko" in header (text treatment, orange color)
8. Pricing card highlight border (`border-2 border-primary`) — single card on page, orange border signals the featured free-tier offer

Accent is NOT used on: plugin card icons, category headings, body text, or any decorative element outside the list above.

Plugin card icons: `text-text` (`#1C1A17`) at 24px — neutral, not orange. Accent reserved for CTAs and featured elements only.
